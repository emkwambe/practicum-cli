# Lesson: cut — Extract Columns From Text

## [WHEN] You need this when:
1. You have a CSV file and need only one column (e.g., usernames).
2. You want to extract usernames from /etc/passwd.
3. A log file has fixed-width fields and you need a specific one.
4. You are processing command output and need a particular field.

## [INTENT] "Give me only this column."

## [CONTEXT]
Many files and command outputs are structured with columns
separated by a delimiter (comma, colon, tab, space).
cut lets you extract specific columns by position or delimiter.
Think of cut as scissors that trim everything except what you need.

## [ACTION] cut -d<delimiter> -f<field_number> <file>
Common patterns:
  cut -d: -f1 /etc/passwd        — get usernames (field 1, colon-delimited)
  cut -d, -f2,3 data.csv         — get columns 2 and 3 from CSV
  cut -c1-10 file.txt             — get first 10 characters per line
  echo "a:b:c" | cut -d: -f2     — extract "b" from piped input

## [EXPECTED RESULT]
Only the specified fields/columns are printed, one per line.
Each line is trimmed to show only your selected fields.

## Try it now:
Run: echo "root:x:0:0:root:/root:/bin/bash" | cut -d: -f1
You should see: root
