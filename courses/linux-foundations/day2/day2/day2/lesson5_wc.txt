# Lesson: wc — Count Lines, Words, and Characters

## [WHEN] You need this when:
1. You want to know how many lines are in a log file.
2. You need to count words in a document.
3. You are checking if a file is empty (0 lines).
4. You want to verify a data file has the expected number of records.
5. You need a quick size check without opening the file.

## [INTENT] "How big is this file?" or "How many lines?"

## [CONTEXT]
wc (word count) gives you three numbers: lines, words, and bytes.
You almost always want just one of these, so use the specific flags.
wc is commonly used at the end of a pipe to count filtered results,
for example: grep "ERROR" log | wc -l (count error lines).

## [ACTION] wc [options] <file>
Common patterns:
  wc file.txt                    — show lines, words, bytes
  wc -l file.txt                 — count lines only
  wc -w file.txt                 — count words only
  wc -c file.txt                 — count bytes
  ls | wc -l                     — count files in current directory
  grep "ERROR" log | wc -l       — count error lines

## [EXPECTED RESULT]
A single number (with the flag) or three numbers (without flags).
The number represents the count of lines, words, or bytes.

## Try it now:
Run: echo -e "one\ntwo\nthree" | wc -l
You should see: 3
