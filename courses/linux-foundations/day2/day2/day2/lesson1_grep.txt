# Lesson: grep — Search Inside Files

## [WHEN] You need this when:
1. A server is throwing errors and you need to find them in log files.
2. You are looking for a specific function name across many source files.
3. You need to filter command output to show only matching lines.
4. You want to check if a config file contains a certain setting.
5. You need to count how many times a pattern appears in a file.

## [INTENT] "Find lines that match a pattern."

## [CONTEXT]
grep stands for "Global Regular Expression Print."
It reads files (or piped input) line by line and prints only
the lines that match your pattern. Everything else is hidden.
Think of grep as a spotlight — it shows only what matters.

## [ACTION] grep <pattern> <file>
Common patterns:
  grep "ERROR" server.log        — find error lines
  grep -i "warning" server.log   — case-insensitive search
  grep -n "TODO" *.py            — show line numbers
  grep -r "password" /etc/       — search recursively in folders
  grep -c "ERROR" server.log     — count matching lines
  grep -v "DEBUG" server.log     — show lines that do NOT match

## [EXPECTED RESULT]
Only the lines containing your pattern are printed.
If no lines match, grep prints nothing and exits silently.

## Try it now:
In Lab mode, run: grep "ERROR" /var/log/syslog
You should see only the error lines from the log file.
