# Lesson: sort — Order Lines Alphabetically or Numerically

## [WHEN] You need this when:
1. You have a list of names and need them in alphabetical order.
2. You want to find the largest files by sorting sizes.
3. Log entries need to be ordered by date or severity.
4. You are preparing data for deduplication with uniq (which requires sorted input).

## [INTENT] "Put these lines in order."

## [CONTEXT]
sort reads lines from a file or pipe and outputs them in order.
By default it sorts alphabetically (A-Z). You can also sort
numerically, in reverse, or by a specific column.
sort is often paired with uniq — sort first, then remove duplicates.

## [ACTION] sort [options] <file>
Common patterns:
  sort names.txt                  — alphabetical order
  sort -n numbers.txt             — numeric order (1, 2, 10 not 1, 10, 2)
  sort -r names.txt               — reverse order (Z-A)
  sort -k2 -t: /etc/passwd        — sort by 2nd field, colon-delimited
  sort -u names.txt               — sort and remove duplicates in one step

## [EXPECTED RESULT]
All lines printed in the specified order.
The original file is NOT modified (sort only prints to screen).

## Try it now:
Run: echo -e "banana\napple\ncherry" | sort
You should see: apple, banana, cherry (one per line).
