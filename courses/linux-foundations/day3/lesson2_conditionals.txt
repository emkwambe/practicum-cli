# Lesson: Conditionals — Making Decisions in Scripts

## [WHEN] You need this when:
1. A script should behave differently based on a condition.
2. You need to check if a file exists before deleting it.
3. You want to validate user input before proceeding.
4. You need to handle errors gracefully in automation.
5. You are writing a backup script that skips if disk is full.

## [INTENT] "If this is true, do that; otherwise, do something else."

## [CONTEXT]
Conditionals let your script make decisions.
The if statement tests a condition and runs code only if true.
Conditions are tested with [ ] (test command) or [[ ]] (bash extended).
Common tests: file exists (-f), directory exists (-d), strings equal (=),
numbers compare (-eq, -lt, -gt).

## [ACTION] if [ condition ]; then ... fi
Common patterns:
  if [ -f "config.txt" ]; then       — file exists?
      echo "Config found"
  else
      echo "Config missing"
  fi

  if [ "$count" -gt 10 ]; then       — number comparison
      echo "More than 10"
  fi

  if [ -z "$name" ]; then            — string is empty?
      echo "Name not set"
  fi

  if [ -d "/backup" ] && [ -w "/backup" ]; then  — combine conditions
      echo "Backup dir exists and is writable"
  fi

## [EXPECTED RESULT]
Only the matching branch executes.
If the condition is false and there is no else, nothing happens.

## Try it now:
  if [ -f /etc/passwd ]; then echo "passwd exists"; else echo "not found"; fi
  if [ 5 -gt 3 ]; then echo "Math works"; fi
