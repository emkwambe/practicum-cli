# Lesson: Secrets & Environments — Safe API Keys in CI

## [WHEN] You need this when:
1. Your workflow needs an API key but you can't commit it to the repo.
2. You need different credentials for staging vs production.
3. Your deploy job needs AWS keys, Docker Hub tokens, or database passwords.

## [INTENT] "Give my workflow access to secrets without exposing them."

## [CONTEXT]
Never commit API keys, passwords, or tokens to your repository.
GitHub provides encrypted Secrets that workflows can access as
environment variables. Secrets are hidden from logs automatically.

Two types:
  Repository secrets    — available to all workflows in the repo
  Environment secrets   — scoped to specific environments (staging, prod)

Environments add approval gates: "require reviewer to deploy to prod."

## [ACTION] Set up secrets
  1. GitHub → repo → Settings → Secrets and variables → Actions
  2. Click "New repository secret"
  3. Name: AWS_ACCESS_KEY (uppercase, underscores)
  4. Value: your actual key
  5. Click "Add secret"

Use in workflow:
  name: Deploy
  on:
    push:
      branches: [main]
  jobs:
    deploy:
      runs-on: ubuntu-latest
      environment: production      # requires approval if configured
      steps:
        - uses: actions/checkout@v4
        - name: Deploy to AWS
          env:
            AWS_KEY: ${{ secrets.AWS_ACCESS_KEY }}
            AWS_SECRET: ${{ secrets.AWS_SECRET_KEY }}
          run: |
            aws configure set aws_access_key_id $AWS_KEY
            aws s3 sync ./dist s3://my-bucket

## Security rules:
  - Secrets are NEVER shown in logs (GitHub auto-masks them)
  - Pull requests from forks CANNOT access secrets by default
  - Rotate secrets if you suspect exposure
  - Use OIDC instead of long-lived keys when possible

## [EXPECTED RESULT]
Your workflow deploys with real credentials without exposing them.
Logs show *** where secrets would have been printed.

## Try it now:
  Add a test secret called MY_SECRET, reference it in a workflow,
  run the workflow, and verify it is masked in the logs.
