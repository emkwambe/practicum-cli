Q1: What does grep -o do differently from plain grep?
A) Outputs the filename
B) Outputs ONLY the matching portion of the line, not the full line
C) Outputs in reverse order
D) Outputs without color
Correct: B

Q2: Why must you sort BEFORE uniq?
A) uniq requires sorted input
B) uniq only removes CONSECUTIVE duplicates, so unsorted input leaves duplicates
C) sort is faster after uniq
D) uniq crashes on unsorted input
Correct: B

Q3: awk '{count[$2]++} END {for (k in count) print k, count[k]}' does what?
A) Counts characters in column 2
B) Groups by column 2 and counts occurrences (like SQL GROUP BY + COUNT)
C) Sorts column 2
D) Prints column 2 twice
Correct: B

Q4: Why use --line-buffered with grep in a pipe after tail -f?
A) It's faster
B) Without it, grep buffers output and you don't see results in real time
C) It enables color
D) It's required by POSIX
Correct: B

Q5: Which sed command removes the first line (header) of a CSV?
A) sed '1d'
B) sed 's/1//'
C) sed '/1/d'
D) sed -n '1p'
Correct: A

Q6: How do you extract ONLY IP addresses from a log file?
A) grep "IP" logfile
B) grep -oE '[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' logfile
C) awk '{print IP}' logfile
D) cut -d. -f1 logfile
Correct: B

Q7: Why does awk -F, fail on CSVs with quoted fields containing commas?
A) awk doesn't support -F
B) awk splits on ALL commas, including those inside quoted fields
C) awk can't read CSV
D) -F only works with single characters
Correct: B

Q8: What makes Unix pipes more memory-efficient than Excel for large files?
A) Pipes use GPU
B) Pipes stream line-by-line (constant memory); Excel loads the entire file into RAM
C) Pipes compress the data
D) No difference
Correct: B

Q9: The regex [0-9]{3,5} matches:
A) Exactly 3 digits
B) Between 3 and 5 consecutive digits
C) 3 or 5 of anything
D) The numbers 3 through 5
Correct: B

Q10: grep -oP '(?<=user=)\w+' does what?
A) Searches for the word "user"
B) Extracts the value AFTER "user=" without including "user=" in the output
C) Prints lines containing user=
D) Replaces user= with nothing
Correct: B

Q11: In sed, the & in 's/[0-9]+/(&)/g' represents:
A) The ampersand character
B) The entire matched text (whatever the regex matched)
C) The last field
D) The line number
Correct: B

Q12: sed -n '/ERROR/p' behaves like which command?
A) sed '/ERROR/d'
B) grep "ERROR" (prints only matching lines)
C) awk '{print ERROR}'
D) cut -d ERROR
Correct: B

Q13: awk '{printf "%-20s %10.2f\n", $1, $3}' produces:
A) Unformatted output
B) Left-aligned $1 (20 chars wide) and right-aligned $3 (float, 2 decimals)
C) Only the first field
D) A syntax error
Correct: B

Q14: awk 'NF != 5 {print NR": "$0}' is useful for:
A) Printing 5 fields
B) Data quality check — finding rows that don't have exactly 5 fields
C) Counting to 5
D) Skipping 5 lines
Correct: B

Q15: The \K operator in grep -oP 'user=\Kw+' does what?
A) Kills the process
B) Resets the match start — everything before \K is required but not output
C) Enables Korean language
D) Skips empty lines
Correct: B
