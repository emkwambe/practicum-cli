# Lesson: sed Advanced — Backreferences, Suppress, and Scripts

## [WHEN] You need this when:
1. You need to rearrange parts of a matched pattern (swap columns).
2. You need to extract and reformat structured fields.
3. You need to run multiple sed operations from a file.
4. You need to suppress output and print only matching lines.

## [INTENT] "Use sed's full power for complex text transformations."

## [CONTEXT]
Beyond basic substitution, sed has three features that make it
a serious data transformation tool: backreferences, suppress mode,
and the transliterate command.

## [ACTION] Advanced sed

### Backreferences — capture and rearrange:
  # \( \) captures a group, \1 \2 \3 refer back to them
  # Swap first and last name:
  echo "Smith, John" | sed 's/\([^,]*\), \(.*\)/\2 \1/'
  # John Smith

  # Extended regex (-E) is cleaner:
  echo "Smith, John" | sed -E 's/([^,]+), (.*)/\2 \1/'
  # John Smith

  # Reformat dates: MM/DD/YYYY → YYYY-MM-DD
  echo "04/17/2026" | sed -E 's|([0-9]{2})/([0-9]{2})/([0-9]{4})|\3-\1-\2|'
  # 2026-04-17

  # Extract domain from email:
  echo "eddy@mpingo.com" | sed -E 's/.*@(.*)/\1/'
  # mpingo.com

  # Wrap matched text:
  echo "Error on line 42" | sed -E 's/([0-9]+)/[\1]/'
  # Error on line [42]

### The & operator — entire match:
  # & represents everything that matched
  echo "1234" | sed 's/[0-9]\+/(&)/'
  # (1234)

  echo "error warning info" | sed -E 's/[a-z]+/[\0]/g'   # GNU sed: \0 = &
  # [error] [warning] [info]

  # Add quotes around numbers:
  echo "value: 42" | sed -E 's/[0-9]+/"&"/g'
  # value: "42"

### Suppress mode (-n) + print (p):
  # Only print lines matching a pattern:
  sed -n '/ERROR/p' log.txt                  # like grep "ERROR"
  sed -n '/^2026-04-17/p' log.txt           # today's entries only
  sed -n '10,20p' file.txt                   # print lines 10-20
  sed -n '1p' file.txt                       # print first line only

  # Combined: extract and print only transformed matches:
  sed -n -E 's/.*user=([^ ]+).*/\1/p' log.txt
  # Only prints the username, only from lines containing user=

### Transliterate (y) — character-by-character replacement:
  # Like tr but inside sed:
  sed 'y/abcdefghijklmnopqrstuvwxyz/ABCDEFGHIJKLMNOPQRSTUVWXYZ/' file
  # Uppercase entire file

  sed 'y/aeiou/AEIOU/' file
  # Uppercase only vowels

  echo "1-2-3" | sed 'y/-/./'
  # 1.2.3

### Script files (-f) — complex multi-step transforms:
  # Create a sed script:
  cat > clean.sed << 'EOF'
  # Remove comments
  /^#/d
  # Remove empty lines
  /^$/d
  # Strip trailing whitespace
  s/[[:space:]]*$//
  # Normalize separators
  s/[[:space:]]\+/,/g
  # Lowercase
  y/ABCDEFGHIJKLMNOPQRSTUVWXYZ/abcdefghijklmnopqrstuvwxyz/
  EOF

  sed -f clean.sed raw_data.txt > clean_data.txt

### Addresses — apply operations to specific lines:
  sed '1s/^/HEADER: /'                     # modify only line 1
  sed '$s/$/ END/'                          # modify only last line
  sed '10,20s/old/new/g'                   # modify lines 10-20 only
  sed '/START/,/END/s/foo/bar/g'           # modify between patterns
  sed '0~2s/^/  /'                          # indent every even line (GNU)

## [EXPECTED RESULT]
sed becomes a full transformation language. Backreferences let you
rearrange data. Suppress mode lets you use sed as a precision extractor.
Script files let you build reusable transform pipelines.

## Technician Tip:
The combo sed -n + backreference + p is the most powerful extraction pattern:
  sed -n -E 's/.*"name": "([^"]+)".*/\1/p' file.json
  → Extracts JSON values without jq (useful when jq isn't installed)

## Common Pain Point:
"Backreferences don't work" → check if using BRE vs ERE:
  BRE (default): \( \) for groups, \1 for reference
  ERE (-E flag):  ( ) for groups, \1 for reference
Always use -E for cleaner syntax.
