# Lesson: Swarm Secrets & Configs — Distribute Sensitive Data Safely

## [WHEN] You need this when:
1. You need to give services database passwords, API keys, TLS certs.
2. Compliance requires encrypted-at-rest credentials.
3. You want to rotate secrets without rebuilding images.
4. You need different config files per environment.

## [INTENT] "Distribute secrets and configs to services without leaks."

## [CONTEXT]
Swarm has built-in secret and config management:
  Secrets — encrypted at rest in Raft store, mounted as files in /run/secrets
  Configs — like secrets but NOT encrypted (for non-sensitive config files)

Both are mounted into containers as files (NOT environment variables).
Both are immutable — to update, create a new version and rotate.

## [ACTION] Secrets and configs

### Create a secret (multiple methods):
  # From stdin:
  echo "supersecret123" | docker secret create db_password -

  # From a file:
  docker secret create tls_cert ./cert.pem

  # List, inspect, remove:
  docker secret ls
  docker secret inspect db_password
  docker secret rm db_password

### Use a secret in a service:
  docker service create \
    --name db \
    --secret db_password \
    -e POSTGRES_PASSWORD_FILE=/run/secrets/db_password \
    postgres:16

  # Inside container, secret is at: /run/secrets/db_password
  # File permissions: 0444, owned by root by default

### Custom path / permissions:
  docker service create \
    --secret source=db_password,target=/etc/myapp/db.txt,uid=1000,mode=0400 \
    --user 1000:1000 \
    myapp

### Configs (same syntax, but unencrypted):
  docker config create nginx_conf ./nginx.conf
  docker service create \
    --name web \
    --config source=nginx_conf,target=/etc/nginx/nginx.conf \
    nginx

### Rotate a secret (immutable, so create+swap):
  echo "newsecret" | docker secret create db_password_v2 -
  docker service update \
    --secret-rm db_password \
    --secret-add source=db_password_v2,target=db_password \
    db
  # Old secret can be removed once nothing uses it
  docker secret rm db_password

## [EXPECTED RESULT]
Inside the container:
  $ ls -la /run/secrets/
  -r--r--r-- 1 root root 14 ... db_password
  $ cat /run/secrets/db_password
  supersecret123

Outside (security):
  $ docker secret inspect db_password
  → metadata only, NOT the value
  $ docker exec <other-container> cat /run/secrets/db_password
  → file doesn't exist (only mounted in services that requested it)

## Try it now:
  docker swarm init
  echo "myDbPass!" | docker secret create db_pass -
  docker secret ls

  docker service create \
    --name db \
    --secret db_pass \
    -e POSTGRES_PASSWORD_FILE=/run/secrets/db_pass \
    --replicas 1 \
    postgres:16-alpine

  # Wait a bit, then check inside:
  sleep 10
  CONTAINER=$(docker ps --filter "name=db" -q)
  docker exec $CONTAINER cat /run/secrets/db_pass

  # Cleanup
  docker service rm db
  docker secret rm db_pass
  docker swarm leave --force

## Technician Tip — secrets vs configs vs env vars:
  Secrets:  encrypted, file-based, for credentials
  Configs:  not encrypted, file-based, for nginx.conf / etc.
  Env vars: visible to anyone with docker inspect — dev only

The PostgreSQL official image supports `_FILE` suffix for env vars:
  POSTGRES_PASSWORD=secret           ← visible in inspect
  POSTGRES_PASSWORD_FILE=/run/secrets/db_pass  ← reads from file (preferred)

Many official images support this pattern. Check the image docs.

## Common Pain Point:
"I updated the secret but app still uses old value" → secrets are
IMMUTABLE. To rotate: create a new secret, update service to use it,
remove old secret. The service must be updated for the change to take effect.

"Secret file is empty / missing" → secret only exists in services that
request it via --secret. Standalone containers cannot access Swarm secrets.

## DCA Exam Note:
Know that secrets are mounted at /run/secrets/<name> by default.
Know that secrets are encrypted at rest in the Raft store.
Know that secrets are tmpfs (in-memory, never on disk in container).
Know that secrets are IMMUTABLE — must create new and swap to rotate.
Know that secrets only work in Swarm services, not docker run.
